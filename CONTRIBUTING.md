# Contributing to Essencium Frontend v2

Thank you for considering contributing to this project!

## Ground Rules

- Please review and follow our [Code of Conduct](./CODE_OF_CONDUCT.md).
- Ensure cross-platform compatibility (Windows, Mac, Linux).
- Create issues for major changes before starting work.
- Keep pull requests focused — one feature or fix per PR.
- Adhere to the coding standards described in [CLAUDE.md](./CLAUDE.md).

## Getting Started

1. Fork the repository
2. Create a new branch: `git checkout -b feat/my-feature`
3. Install dependencies: `pnpm install`
4. Make your changes
5. Run lint and type-check: `pnpm lint && pnpm build`
6. Commit using [Conventional Commits](https://www.conventionalcommits.org/)
7. Open a pull request with a clear description

## Commit Messages

We use [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/):

- `feat:` — new feature
- `fix:` — bug fix
- `chore:` — maintenance, tooling
- `docs:` — documentation only
- `refactor:` — code change without feature or fix
- `test:` — adding or updating tests

First line max 72 characters. Reference issues after the first line.

## Reporting Bugs

For security vulnerabilities, **do not** open a public issue. Email [essencium@frachtwerk.de](mailto:essencium@frachtwerk.de) directly.

For all other bugs, open a GitHub issue using the bug report template.

## Code Review

All pull requests require at least one review before merging. Expect feedback within a reasonable timeframe via pull request comments.
